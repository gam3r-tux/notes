Certificaciones recomendadas
Ruta sugerida para devs
1️⃣ Platform Developer I (PDI) — base fundamental. Por aquí empieza todo dev.
2️⃣ Platform Developer II (PDII) — avanzado: patterns, LWC, integraciones complejas.
3️⃣ JavaScript Developer I — si quieres enfocarte en LWC / frontend.
4️⃣ Integration Architecture Designer — para el track de arquitecto.
Preparar certificación PDI ↗
⏱ Tiempo total estimado (estudio consistente): 4–6 meses para estar listo para PDI. Como Java dev, las fases 0 y 1 te tomarán menos tiempo del promedio.


Aquí tienes el roadmap completo. Algunas notas importantes dado tu background en Java:

**Tu ventaja principal está en Fase 1 (Apex).** La sintaxis de Apex es deliberadamente similar a Java — clases, interfaces, colecciones, excepciones, tipado fuerte. Lo que te costará más no es la sintaxis sino el cambio de mentalidad: en Salesforce existe un sistema de *Governor Limits* que limita cuántas operaciones puedes hacer por transacción (máximo 100 queries SOQL, 150 DML statements, etc.). Esto obliga a escribir código "en bulto" (*bulkification*), que es el patrón más importante que debes dominar.

**Trailhead es el recurso #1.** La plataforma de aprendizaje oficial de Salesforce (trailhead.salesforce.com) es gratuita, gamificada y tiene entornos reales para practicar. Es el equivalente a la documentación oficial pero interactiva.

**Orden sugerido de recursos:**
- Empieza con un **Developer Org gratuita** (signup en developer.salesforce.com)
- Completa los trails de Trailhead en el orden del roadmap
- Practica con **Scratch Orgs** desde VS Code cuando llegues a la fase de herramientas
- Para la certificación PDI, usa los exámenes de práctica oficiales y los trails específicos de preparación
